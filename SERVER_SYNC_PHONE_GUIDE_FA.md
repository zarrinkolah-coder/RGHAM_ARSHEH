# اتصال نسخه نصب‌شده گوشی به سرور — نسخه ۲٫۲

## چرا قبلاً اطلاعات جدید نمایش داده نمی‌شد؟

GitHub Pages فقط فایل‌های ثابت PWA را منتشر می‌کند و `server.py` یا SQLite را اجرا نمی‌کند. بنابراین تا زمانی که سرور Python/Docker در یک آدرس HTTPS جداگانه اجرا نشود و آن آدرس داخل برنامه ثبت نگردد، اطلاعات فقط در حافظه همان گوشی باقی می‌ماند.

## بخش اول: تنظیم سرور

سرور باید روی یک میزبان Python یا Docker اجرا شود و یک آدرس HTTPS مانند زیر داشته باشد:

```text
https://ragham-server.example.com
```

دو متغیر محیطی زیر را روی میزبان سرور تنظیم کنید:

```text
RAGHAM_CORS_ORIGINS=https://zarrinkolah-coder.github.io
RAGHAM_API_TOKEN=یک-کلید-طولانی-و-محرمانه
```

در `RAGHAM_CORS_ORIGINS` فقط مبدا GitHub Pages نوشته می‌شود و مسیر `/RGHAM_ARSHEH/` اضافه نمی‌شود.

پس از راه‌اندازی سرور، این آدرس باید در مرورگر پاسخ JSON بدهد:

```text
https://ragham-server.example.com/api/health
```

اگر `RAGHAM_API_TOKEN` فعال باشد، بازکردن مستقیم این آدرس در مرورگر ممکن است خطای 401 بدهد؛ این طبیعی است و برنامه کلید را در Header ارسال می‌کند.

## بخش دوم: تنظیم برنامه روی گوشی

1. نسخه جدید را در GitHub جایگزین و GitHub Pages را منتشر کنید.
2. برنامه نصب‌شده را باز کنید و با حساب مدیر وارد شوید.
3. از نوار بالا روی «اتصال سرور» بزنید.
4. تیک «فعال‌کردن اتصال به سرور» را روشن کنید.
5. آدرس HTTPS سرور را بدون `/api/state` وارد کنید.
6. کلید تعریف‌شده در `RAGHAM_API_TOKEN` را وارد کنید.
7. روی «ذخیره و اتصال» بزنید.
8. پیام «اتصال ذخیره شد و اطلاعات سرور دریافت گردید» باید نمایش داده شود.

پس از آن برنامه هر ۸ ثانیه نسخه جدید اطلاعات را بررسی می‌کند و تغییرات ثبت‌شده را به SQLite سرور می‌فرستد.

## نکات مهم

- آدرس `https://zarrinkolah-coder.github.io/RGHAM_ARSHEH/` آدرس PWA است، نه آدرس سرور داده.
- سرور باید HTTPS داشته باشد؛ یک برنامه HTTPS نمی‌تواند به سرور HTTP معمولی متصل شود.
- کلید API را داخل فایل‌های GitHub قرار ندهید.
- برای نگهداری SQLite روی میزبان ابری، از دیسک دائمی یا Volume استفاده کنید.
