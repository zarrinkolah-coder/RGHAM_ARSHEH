# اصلاح پیام قطع سرور در نسخه نصب‌شده از GitHub Pages

نسخه 2.1.0 هنگام اجرا روی دامنه `github.io` به‌صورت خودکار وارد «حالت محلی دستگاه» می‌شود.

در این حالت:

- برنامه برای اتصال به `/api/state` تلاش تکراری نمی‌کند.
- پیام «ارتباط با سرور قطع است» نمایش داده نمی‌شود.
- همه تغییرات در حافظه همان گوشی ذخیره می‌شوند.
- پشتیبان‌گیری و بازیابی JSON مدیر فعال است.
- برای انتقال اطلاعات به گوشی دیگر باید از پشتیبان JSON استفاده شود.

## انتشار اصلاح در GitHub

همه فایل‌های این بسته را در ریشه مخزن جایگزین کنید، سپس در GitHub Desktop:

1. `Commit to main`
2. `Push origin`
3. صبر کنید GitHub Pages دوباره Deploy شود.

فایل‌های مهم در ریشه مخزن:

- `index.html`
- `app.js`
- `styles.css`
- `manifest.webmanifest`
- `service-worker.js`
- `icons/`

## بروزرسانی نسخه نصب‌شده روی گوشی

پس از Deploy، برنامه را کاملاً ببندید و دوباره باز کنید. اگر پیام «نسخه جدید برنامه آماده است» نمایش داده شد، دکمه بروزرسانی را بزنید. اگر نسخه قبلی همچنان باقی ماند، برنامه نصب‌شده را حذف و دوباره از سایت یا بسته جدید PWABuilder نصب کنید.

## حالت سروری

هنگام اجرای پروژه با `server.py`، برنامه همچنان به SQLite سرور متصل می‌شود و همگام‌سازی مرکزی فعال است. برای اجبار حالت سروری می‌توان `?mode=server` را به نشانی افزود.
