# نسخه سروری PWA نرم‌افزار رقم

این بسته، نسخه فعلی نرم‌افزار رقم را از حالت فایل HTML مستقل به یک برنامه قابل اجرا روی سرور تبدیل می‌کند. اطلاعات برنامه در پایگاه داده مرکزی SQLite ذخیره می‌شود و همه مرورگرها از طریق API با همان پایگاه داده همگام می‌شوند.

## اجرای سریع روی لینوکس

پیش‌نیاز: Python 3.10 یا جدیدتر.

```bash
unzip ragham_pwa_github_ready_v2.zip
cd ragham_pwa_github_ready_v2
chmod +x start_linux.sh
./start_linux.sh
```

سپس در مرورگر باز کنید:

```text
http://IP-SERVER:8080
```

پورت دلخواه:

```bash
RAGHAM_PORT=9000 ./start_linux.sh
```

## اجرای سریع روی ویندوز سرور

فایل ZIP را Extract کنید و `start_windows.bat` را اجرا کنید. برنامه به‌صورت پیش‌فرض روی پورت 8080 اجرا می‌شود.

## اجرا با Docker

```bash
docker compose up -d --build
```

سپس برنامه از آدرس زیر در دسترس است:

```text
http://IP-SERVER:8080
```

داده‌ها در Volume با نام `ragham_data` نگهداری می‌شوند و با حذف یا بروزرسانی کانتینر از بین نمی‌روند.

## محل پایگاه داده

در اجرای مستقیم:

```text
data/ragham.sqlite3
```

مسیر سفارشی:

```bash
RAGHAM_DATA_DIR=/var/lib/ragham ./start_linux.sh
```

## پشتیبان‌گیری

```bash
python3 backup_db.py
```

فایل پشتیبان در پوشه `backups` ایجاد می‌شود. خود سرور نیز پیش از تغییرات، نسخه‌های قبلی داده را داخل جدول پشتیبان پایگاه داده نگه می‌دارد.

## انتقال فایل JSON موجود به سرور

```bash
python3 import_json.py /path/to/narmafzar-ragham-data.json
```

اگر پایگاه داده سرور خالی باشد، اولین مرورگری که برنامه را باز می‌کند اطلاعات فعلی ذخیره‌شده در همان مرورگر را به سرور منتقل می‌کند. برای انتقال کنترل‌شده، روش `import_json.py` مطمئن‌تر است.

## محافظت اولیه با نام کاربری و رمز کلی

برای جلوگیری از بازشدن عمومی برنامه، می‌توانید یک نام کاربری و رمز کلی روی کل سایت فعال کنید:

```bash
RAGHAM_BASIC_USER=ragham RAGHAM_BASIC_PASSWORD='CHANGE-ME' ./start_linux.sh
```

این رمز جدا از کارت ورود مدیر و کاربران داخل نرم‌افزار است.

## اتصال دامنه و HTTPS

نمونه تنظیم Nginx در فایل `nginx-ragham.conf.example` قرار دارد. نام دامنه را جایگزین `example.com` کنید و SSL را روی Nginx یا پنل هاست فعال کنید.

## اجرای دائمی با systemd

نمونه سرویس در `ragham.service.example` قرار دارد. مسیرها و نام کاربر لینوکس را متناسب با سرور تغییر دهید.

## نکته امنیتی مهم

این بسته نسخه سروری PWA است: ذخیره‌سازی مرکزی، همگام‌سازی چند مرورگر، SQLite، پشتیبان‌گیری و استقرار روی سرور فعال شده‌اند. ورود کارت کاربران و کنترل دسترسی فعلی هنوز عمدتاً در رابط مرورگر اجرا می‌شود و کدهای آزمایشی مدیر `1` و کاربران `2` نیز فعال هستند. برای انتشار مستقیم روی اینترنت، فعلاً از HTTPS و محافظت کلی Basic Auth یا VPN استفاده کنید. احراز هویت واقعی سمت سرور، نشست امن، رمزنگاری کدها و کنترل دسترسی ردیفی باید در مرحله بعد تکمیل شوند.


## پشتیبان‌گیری و بازیابی JSON توسط مدیر

پس از ورود با حساب مدیر، دو دکمه در نوار بالای برنامه نمایش داده می‌شود:

- **ذخیره پشتیبان JSON**: همه کالاها، کاربران، طرف‌حساب‌ها، فاکتورها، تراکنش‌ها، گردش انبار، تنظیمات و تاریخچه اصلاحات را در یک فایل JSON دانلود می‌کند.
- **بازیابی پشتیبان**: فایل JSON را بررسی می‌کند، خلاصه تعداد رکوردها را نشان می‌دهد و پس از تأیید، داده‌های پایگاه SQLite سرور را جایگزین می‌کند.

پیش از هر بازیابی، برنامه به‌صورت خودکار یک فایل JSON از وضعیت فعلی دانلود می‌کند. همچنین خود سرور نسخه قبلی را در جدول `state_backups` پایگاه SQLite نگهداری می‌کند. حداکثر تعداد نسخه‌های داخلی با متغیر محیطی `RAGHAM_BACKUP_LIMIT` قابل تنظیم است.

حجم فایل بازیابی در رابط برنامه حداکثر ۲۵ مگابایت است. برای تغییر سقف درخواست سرور، متغیر `RAGHAM_MAX_BODY` را تنظیم کنید.

> توجه امنیتی: در نسخه ۱.۱ دکمه‌های پشتیبان و بازیابی فقط در نمای مدیر نمایش داده می‌شوند، اما احراز هویت و مجوزدهی کاملاً سمت سرور هنوز جزو مرحله امنیتی بعدی است. تا آن زمان برنامه را پشت HTTPS و Basic Auth یا VPN اجرا کنید.
